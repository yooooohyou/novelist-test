## 一、项目概览

这是一个 AI 辅助中文小说创作工作站，集成 7 个 Claude Code Skills，支持从参考作品拆解到成稿输出的完整仿写流程。当前在写作品为《洞悉天机三千年，一不小心无敌了》（仿写《顶级气运，悄悄修炼千年》），已完成第一卷 10 章，规划 100 章 10 卷。

## 二、仿写工作流 6 阶段详解

### Phase 1：拆书分析

| 项目 | 说明 |
|------|------|
| **目标** | 从参考作品中提取可仿写的风格规范 |
| **输入** | `references/` 下的参考小说原文 |
| **输出** | `02-风格参考手册.md` |
| **方法** | 人工 + AI 协同分析，覆盖 11 个维度 |

**分析的 11 个维度：**

1. **核心风格标签** — 系统流/扮猪吃虎/苦修龟缩/好感度游戏/轻松搞笑
2. **叙事视角** — 第三人称有限，高密度内心独白，极口语化
3. **语言特征清单** — 必用项（网络吐槽/自嘲/反差独白/短句爆发）+ 禁用项（华丽辞藻/大段环境描写/诗词/文言）
4. **系统面板格式** — 属性面板/选择系统/好感度/人际关系的精确排版
5. **节奏模式** — 时间跳跃（微/中/大）、延迟满足、章节节奏公式（开场10%+触发20%+反应30%+行动25%+结果15%）
6. **爽点设计** — 6 种类型：扮猪吃虎/属性碾压/选择奖励/好感度上涨/秒杀战斗/低调失败喜剧
7. **角色设计范式** — 主角模板（前世记忆/苦修/表里不一），配角分 5 种类型
8. **章尾钩子** — 6 种类型轮换：悬念/危机/反转/期待/搞笑/新角色
9. **世界观搭建** — 不集中设定，随剧情自然展开
10. **写作禁忌** — 8 条红线（不让主角主动找麻烦/不写冗长打斗/不煽情等）
11. **仿写质量对标** — 让读者笑/爽/翻页、不像 AI 写、像老练网文写手

**本阶段关键产出示例：** 当前项目根目录的 `02-风格参考手册.md` 即为 Phase 1 成果，基于《顶级气运，悄悄修炼千年》拆解。

### Phase 2：骨架搭建

| 项目 | 说明 |
|------|------|
| **目标** | 生成完整的故事骨架 |
| **调用技能** | `chinese-novelist`（第一阶段 + 第二阶段） |
| **输入** | 用户需求 + 风格参考手册 |
| **输出** | `00-大纲.md` + `01-人物档案.md` |

**执行步骤：**

1. **5 问确认**（通过 AskUserQuestion 逐一询问）：
   - 题材和世界观类型
   - 主角基本设定
   - 主角性格特点
   - 核心冲突/矛盾
   - 章节数量规划

2. **生成大纲** `00-大纲.md`：
   - 使用 `outline-template.md` 模板
   - 含基本信息、TODO List、分卷章节规划表
   - 全书悬念线设计、字数统计

3. **生成人物档案** `01-人物档案.md`：
   - 使用 `character-template.md` + `character-building.md` 指导
   - 主角完整档案（基本信息/性格核心/背景故事/行为模式）
   - 反派、配角、NPC 档案
   - 人际关系网 + 好感度进展表

4. **用户确认** — 展示摘要，等待用户审阅通过

**当前项目实例：** 大纲 100 章 10 卷全部规划完成，人物档案含主角韩渊 + 2 女性角色 + 4 核心配角 + 3 天命之子 + 3 反派 + 3 弟子 + 2 重要 NPC。

### Phase 3：逐章创作

| 项目 | 说明 |
|------|------|
| **目标** | 逐章完成小说正文 |
| **调用技能** | `chinese-novelist`（第三阶段） |
| **关键约束** | **必须串行执行**，确保剧情连贯 |
| **字数要求** | 2500-3500 字/章（短剧爽文节奏） |

**每章创作流程（14 步）：**

```
┌─ 写前准备 ──────────────────────────────────────────┐
│ 1. 读取大纲 TODO List + 上章摘要                      │
│ 2. 标记本章为"进行中"                                 │
│ 3. 设计开头钩子（前20%必须有即时冲突）                  │
│ 4. 规划 3-5 个场景                                    │
└──────────────────────────────────────────────────────┘
           ↓
┌─ 正文撰写 ──────────────────────────────────────────┐
│ 5. 创建章节文件（chapter-template.md 模板）            │
│ 6. 撰写正文 3000-5000 字                              │
│ 7. 设置结尾钩子                                       │
└──────────────────────────────────────────────────────┘
           ↓
┌─ 质量保障 ──────────────────────────────────────────┐
│ 8. 字数检查（check_chapter_wordcount.py）             │
│    → 低于 3000 字必须扩充                              │
│ 9. 连贯性检查（consistency.md 自检清单）               │
│10. 深度润色（去AI味初筛）                              │
│    → 去除过度修饰词、减少抽象陈述                       │
│    → 打破四字格律、增加口语化                           │
│    → 优化节奏、细节具象化                               │
│11. 再次字数检查                                        │
└──────────────────────────────────────────────────────┘
           ↓
┌─ 收尾归档 ──────────────────────────────────────────┐
│12. 生成章节摘要（300-500 字）                          │
│13. 更新大纲 TODO 为"完成"                              │
│14. 进入下一章循环                                      │
└──────────────────────────────────────────────────────┘
```

**依赖的参考文档（11 份）：**

| 文档 | 用途 |
|------|------|
| `chapter-guide.md` | 10 种强力开头技巧 + 章节结构（开头钩子20% + 发展推进50-60% + 高潮15-20% + 结尾钩子5-10%） |
| `hook-techniques.md` | 10 种经典结尾钩子 + 5 级悬念强度 + 4 大禁忌 |
| `content-expansion.md` | 7 种扩充方法（场景细节/人物内心/对话扩展/感官体验/次要情节/节奏放慢/环境烘托） |
| `dialogue-writing.md` | 对话 3 大核心原则 + 角色声音区分 5 维度 + 潜台词 4 种技巧 |
| `consistency.md` | 写前必读 3 步 + 人物状态跟踪（位置/情绪/关系/能力）+ 悬念线延续 |
| `character-building.md` | 人物深度塑造 5 种技巧（矛盾性/侧面揭示/声音独特性/动机合理化/缺陷致命化） |
| `plot-structures.md` | 6 种情节结构模板（三幕式/英雄之旅/悬疑/言情/惊悚/反转） |
| `quality-checklist.md` | 8 维度评分自检 |
| `chapter-template.md` | 章节文件模板 |
| `outline-template.md` | 大纲模板 |
| `character-template.md` | 人物档案模板 |

**三大黄金法则：** 展示而非讲述、冲突驱动剧情、悬念承上启下

### Phase 4：润色去 AI

| 项目 | 说明 |
|------|------|
| **目标** | 系统性去除 AI 生成痕迹 |
| **调用技能** | `humanizer-zh` |
| **特点** | **可并行处理**多个章节 |
| **达标线** | >= 35/50 分 |

**24 种检测模式（4 大类）：**

| 类别 | 模式数 | 典型模式 |
|------|--------|----------|
| 内容模式 | 6 | 过度强调意义/遗产、宣传语言、模糊归因、以-ing结尾的肤浅分析 |
| 语言语法模式 | 6 | AI高频词汇、系动词回避、否定式排比、三段式过度使用、刻意换词 |
| 风格模式 | 6 | 破折号过度、粗体过度、内联标题列表、表情符号 |
| 交流+填充 | 6 | 协作交流痕迹、谄媚语气、填充短语、过度限定、通用积极结论 |

**5 条核心规则：**
1. 删除一切填充短语
2. 打破公式化结构
3. 变化句子节奏
4. 信任读者智商（不解释隐喻）
5. 删除"金句"式总结

**评分体系（5 维度各 10 分）：**
- 直接性 — 不绕弯子
- 节奏 — 长短句交替自然
- 信任度 — 不过度解释
- 真实性 — 有具体个性
- 精炼度 — 无废话

**注入灵魂：** 不仅去除 AI 模式，还要注入真实个性——有观点、变化节奏、承认复杂性、适当用"我"、允许混乱、对感受具体。

### Phase 5：质量审查

| 项目 | 说明 |
|------|------|
| **目标** | 多维度质量把关 |
| **达标线** | >= 60/80 分 |
| **审查级别** | P0 硬伤 / P1 可优化 / P2 外观清理 |

**8 维度 80 分制评分（quality-checklist.md）：**

| 维度 | 满分 | 评估要点 |
|------|------|----------|
| 开头吸引力 | 10 | 前 20% 是否有即时冲突/重大事件/情感冲击 |
| 情节推进 | 10 | 每个场景是否推动故事前进 |
| 人物塑造 | 10 | 行为是否符合性格，是否有成长 |
| 对话质量 | 10 | 是否有目的、简洁、区分角色声音 |
| 悬念设置 | 10 | 章末是否有有效钩子 |
| 节奏控制 | 10 | 张弛有度，不拖沓不跳跃 |
| 展示而非讲述 | 10 | 用动作/对话代替陈述 |
| 语言质量 | 10 | 文字流畅、风格一致 |

**三重检查：**
1. **评分检查** — 8 维度逐项评分，>= 60 可交付，>= 70 优秀
2. **连贯性检查** — 时间线一致、人物行为一致、伏笔呼应、悬念延续
3. **风格对比** — 与 `02-风格参考手册.md` 对照，确保风格不跑偏

**当前项目实例：** `03-审查报告.md` 对前 10 章审查发现：
- **P0 硬伤（5 个）：** 天机隐匿消耗模型前后不一致、预警窗口 3 日 vs 实际 7 日矛盾、屏蔽符文修复成本矛盾、"定时失效" vs "手动触发"冲突、摘要与正文矛盾
- **P1 可优化（5 个）：** 待改善项
- **P2 外观清理（2 个）：** 格式和细节

### Phase 6：输出成稿

| 项目 | 说明 |
|------|------|
| **目标** | 生成可交付的最终文档 |
| **调用技能** | `docx` + `build_reader.py` |

**三种输出格式：**

1. **合并文本** — 将所有章节合并为完整小说文档
2. **Word 文档** — 使用 `docx` 技能（docx-js 库），支持目录/页眉页脚/分页/样式
3. **HTML 阅读器** — 使用 `build_reader.py`，生成自包含的 `reader.html`
   - 扫描所有 `第XX章.md` → 解析标题/概要/备注/段落
   - 转换大纲和人物档案为 HTML
   - 嵌入封面图（base64）
   - 单文件分发，无外部依赖
